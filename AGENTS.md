# Kuro AI — Agent Instructions

## Documentation First Rule

Before writing production code:

- Read all documentation first (onboarding, architecture, roadmap, memory, decisions, conventions)
- Populate docs with initial content before starting work
- Summarize project understanding
- Identify unknowns
- Produce implementation plan

Implementation begins only after project context is established.

A feature is not complete until:

- Code is updated
- Documentation is updated
- Memory is updated
- Roadmap is updated if priorities changed

## Global Rules

- Read project docs before coding
- Create docs if missing
- Plan before implementation
- Verify with tests and lint
- Keep documentation synchronized with code
- Never commit API keys or secrets
- Follow existing code conventions (see conventions.md)
